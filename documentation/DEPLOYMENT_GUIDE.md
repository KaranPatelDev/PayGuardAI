| Backend API | Render | https://payguardai.onrender.com |
| Frontend | Vercel | (your Vercel URL) |

## Render Deployment (Backend)

### Blueprint
The `render.yaml` at the project root defines the backend service:

```yaml
services:
  - type: web
    name: payguard-api
    runtime: python
    plan: free
    buildCommand: pip install -r backend/requirements.txt
    startCommand: cd backend && uvicorn server:app --host 0.0.0.0 --port $PORT
```

### Steps
1. Push code to GitHub
2. Go to https://dashboard.render.com ‚Üí New ‚Üí Blueprint
3. Connect your GitHub repo
4. Render reads `render.yaml` and creates the service
5. Set environment variables in the Render dashboard:
   - `DATABASE_URL` ‚Äî Your Neon connection string (see below)
   - `JWT_SECRET` ‚Äî Auto-generated by Render or set your own
   - `CORS_ORIGINS` ‚Äî Your Vercel URL (e.g., `https://your-app.vercel.app`)
   - `DEMO_PASSWORD` ‚Äî `demo123`
6. Deploy

### Environment Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `DATABASE_URL` | Neon PostgreSQL connection string | `postgresql+asyncpg://user:pass@host/db?sslmode=require` |
| `JWT_SECRET` | Secret for JWT token signing | Auto-generated by Render |
| `JWT_ALGORITHM` | JWT algorithm | `HS256` |
| `JWT_EXPIRE_MINUTES` | Token expiry in minutes | `10080` (7 days) |
| `CORS_ORIGINS` | Allowed CORS origins | `https://your-app.vercel.app` or `*` |
| `LOG_LEVEL` | Logging level | `INFO` |
| `LOG_DIR` | Log file directory | `/tmp/logs` |
| `DEMO_EMAIL` | Demo account email | `demo@payguard.ai` |
| `DEMO_PASSWORD` | Demo account password | `demo123` |

### SSL Configuration (Neon)
The backend automatically handles Neon's SSL requirements by stripping `sslmode` and `channel_binding` query parameters from the connection URL and configuring SSL via `connect_args`. No manual SSL configuration needed.

## Vercel Deployment (Frontend)

### Steps
1. Go to https://vercel.com ‚Üí Sign in with GitHub
2. Click "Add New Project"
3. Import the GitHub repo
4. Configure:
   - **Framework Preset**: Create React App
   - **Root Directory**: `frontend`
   - **Build Command**: `yarn install && yarn build`
   - **Output Directory**: `build`
5. Add Environment Variable:
   - `REACT_APP_BACKEND_URL` = `https://payguardai.onrender.com`
6. Click Deploy

### SPA Routing
The `frontend/vercel.json` handles client-side routing:
```json
{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```

### Important Notes
- `REACT_APP_BACKEND_URL` is baked into the build at compile time
- Changing it requires a redeploy on Vercel
- After frontend deploy, update `CORS_ORIGINS` on Render to match the Vercel URL

## Neon Database

### Setup
1. Create account at https://neon.tech
2. Create a new project
3. Copy the connection string from the dashboard
4. The connection string looks like:
   ```
   postgresql://user:pass@ep-xxx.us-east-1.aws.neon.tech/neondb?sslmode=require
   ```
5. For the backend, prepend `+asyncpg` to the protocol:
   ```
   postgresql+asyncpg://user:pass@ep-xxx.us-east-1.aws.neon.tech/neondb?sslmode=require&channel_binding=require
   ```

### Auto-Table Creation
The backend automatically creates all tables on startup via SQLAlchemy's `Base.metadata.create_all`. No manual migration needed.

### Demo Data Seeding
Demo data is seeded automatically on first startup. The seed is idempotent ‚Äî it skips if the demo user already exists.

## Production Checklist
- [ ] Strong `JWT_SECRET` set
- [ ] HTTPS-only access (Render and Vercel provide this by default)
- [ ] `CORS_ORIGINS` restricted to your Vercel URL (not `*` in production)
- [ ] Neon database credentials secured
- [ ] Environment variables configured outside source control
- [ ] Frontend build points to production backend URL
- [ ] Health check returns `{ "app": "PayGuard AI", "status": "ok" }`
- [ ] Neon plan limits reviewed (connection count, storage)

## Troubleshooting
- **401 errors**: Verify token storage and `JWT_SECRET` consistency.
- **CORS errors**: Verify `CORS_ORIGINS` matches the frontend URL exactly.
- **Database connection failure**: Verify `DATABASE_URL` format (must use `postgresql+asyncpg://`), check Neon is active, verify SSL.
- **Empty dashboard**: Verify user has customers/invoices or trigger seed data.
- **AI/OCR failures**: Verify provider credentials and upload file type/size.
- **Render cold start**: Free tier services sleep after inactivity; first request may take 30-60 seconds.
- **Build memory error**: Use `NODE_OPTIONS='--max_old_space_size=4096'` and disable source maps.
