## What to Try

Here are some recommended actions to explore the platform:

### 1. Dashboard Overview
Check the dashboard for a real-time summary of receivables, overdue amounts, and cash flow trends.

### 2. Risk Scoring
Navigate to the **Customers** tab and review the risk scores. Notice how `Verma Metal Crafts` has a high risk score due to repeated late payments.

### 3. AI Follow-up Generation
Go to **Invoices → INV-2026-003** (Verma Metal Crafts) and generate an AI follow-up:

:::tip
Try different tones (professional, friendly, firm) and languages (English, Hindi, Gujarati) to see how the AI adapts the message.
:::

### 4. Record a Payment
Mark `INV-2026-002` as partially paid (₹40,000) and observe how the dashboard updates.

### 5. Cashflow Forecast
View the **Cashflow** tab to see the 30-day forecast based on historical payment patterns.

### 6. Generate Reports
Export an overdue report as PDF from the **Reports** section.

## Resetting Demo Data

If you want to reset the demo account to its original state:

```bash
curl -X POST https://api.payguard.ai/api/v1/demo/reset \
  -H "Authorization: Bearer $PAYGUARD_TOKEN"
```

:::caution
This will delete all changes made during your session and restore the original sample data.
:::

## Limitations

The demo account has the following restrictions:

- Maximum 10 customers
- Maximum 50 invoices
- AI follow-ups limited to 20 per day
- No real payment processing (Razorpay is in test mode)
- Data resets every 24 hours
