# Environment Variables

All configuration is managed via environment variables. Never commit `.env` files to version control.

---

## Backend Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `DATABASE_URL` | Yes | — | PostgreSQL connection string with asyncpg driver |
| `DB_NAME` | No | — | Database name (for reference) |
| `JWT_SECRET` | Yes | — | Secret key for signing JWT tokens |
| `JWT_ALGORITHM` | No | `HS256` | JWT signing algorithm |
| `JWT_EXPIRE_MINUTES` | No | `10080` | Access token expiry in minutes (default: 7 days) |
| `CORS_ORIGINS` | No | `*` | Comma-separated allowed origins, or `*` for all |
| `EMERGENT_LLM_KEY` | Yes | — | API key for Claude LLM access via Emergent |
| `DEMO_EMAIL` | No | `demo@payguard.ai` | Demo account email (auto-seeded) |
| `DEMO_PASSWORD` | No | `demo123` | Demo account password (auto-seeded) |
| `LOG_LEVEL` | No | `INFO` | Logging level (`DEBUG`, `INFO`, `WARNING`, `ERROR`) |
| `LOG_DIR` | No | `./logs` | Directory for log files |
| `SMTP_HOST` | No | `smtp.gmail.com` | SMTP server hostname |
| `SMTP_PORT` | No | `587` | SMTP server port |
| `SMTP_USER` | No | — | SMTP username (email address) |
| `SMTP_PASS` | No | — | SMTP password or app password |
| `FRONTEND_URL` | No | `http://localhost:3000` | Frontend URL for password reset links |

### Examples

**Local development:**

```env
DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/payguard_ai
JWT_SECRET=dev-secret-change-me
CORS_ORIGINS=*
EMERGENT_LLM_KEY=sk-emergent-xxxx
```

**Production:**

```env
DATABASE_URL=postgresql+asyncpg://neondb_owner:xxxx@ep-xxxxx.ap-south-1.aws.neon.tech/payguard_ai?sslmode=require
JWT_SECRET=<random-64-char-hex-string>
CORS_ORIGINS=https://app.payguard.ai
EMERGENT_LLM_KEY=sk-emergent-xxxx
FRONTEND_URL=https://app.payguard.ai
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-16-char-app-password
```

## Frontend Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `REACT_APP_BACKEND_URL` | Yes | `http://localhost:8000` | Backend API base URL |

:::caution
Frontend env vars are baked into the build at compile time. Changing them requires a rebuild and redeploy.
:::

### Examples

**Local development:**

```env
REACT_APP_BACKEND_URL=http://localhost:8000
```

**Production:**

```env
REACT_APP_BACKEND_URL=https://your-app.up.railway.app
```

## Security Notes

- **Never commit `.env` files** to version control. The `.gitignore` already excludes them.
- **Generate a strong `JWT_SECRET`** for production:

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

- **Rotate secrets regularly** — if a key is compromised, regenerate and update all deployments.
- **Use SSL** — always set `DATABASE_URL` with `sslmode=require` in production.
- **Restrict CORS** — set `CORS_ORIGINS` to your frontend domain, not `*`.
- **Use App Passwords** — for Gmail SMTP, never use your main password. Use a [Google App Password](https://myaccount.google.com/apppasswords).
