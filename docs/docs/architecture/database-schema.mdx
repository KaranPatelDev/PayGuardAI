# Database Schema

PayGuard AI uses PostgreSQL (Neon serverless) with 6 tables organized in a star-like pattern centered around the `users` table.

## Table Descriptions

### `users`

The core identity table. Every other table references this via `user_id`.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | Unique user identifier |
| `full_name` | TEXT | NOT NULL | User's display name |
| `email` | TEXT | UNIQUE, NOT NULL | Login email address |
| `phone` | TEXT | | Phone number (optional) |
| `business_name` | TEXT | NOT NULL | Company or trade name |
| `business_type` | TEXT | | Industry or business category |
| `gst_number` | TEXT | | GST registration number |
| `password_hash` | TEXT | NOT NULL | bcrypt-hashed password |
| `created_at` | TEXT | NOT NULL | ISO 8601 UTC timestamp |
| `updated_at` | TEXT | NOT NULL | ISO 8601 UTC timestamp |

### `settings`

Per-user business preferences. One-to-one relationship with `users`.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | Settings record ID |
| `user_id` | UUID | FK â†’ users, UNIQUE | Owner user |
| `default_payment_terms` | INT | DEFAULT 30 | Default invoice payment terms (days) |
| `default_followup_tone` | TEXT | DEFAULT 'Professional' | AI message tone preference |
| `default_reminder_days` | INT[] | DEFAULT {-3,0,3,7,15,30} | Days before/after due for reminders |
| `reminder_channels` | TEXT[] | DEFAULT {'WhatsApp','Email'} | Preferred notification channels |
| `currency` | TEXT | DEFAULT 'INR' | Display currency |
| `ai_provider` | TEXT | DEFAULT 'claude-sonnet-4-6' | AI model to use |
| `updated_at` | TEXT | NOT NULL | Last modification timestamp |

### `invoices`

All invoices created by a user for their customers.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | Invoice identifier |
| `user_id` | UUID | FK â†’ users | Owner user |
| `customer_id` | UUID | FK â†’ customers | Associated customer |
| `invoice_number` | TEXT | NOT NULL | User-facing invoice number |
| `invoice_date` | TEXT | NOT NULL | Date of issue (ISO 8601) |
| `due_date` | TEXT | NOT NULL | Payment due date |
| `amount` | FLOAT | NOT NULL | Pre-tax amount |
| `tax_amount` | FLOAT | DEFAULT 0 | Tax amount |
| `total_amount` | FLOAT | DEFAULT 0 | amount + tax_amount |
| `paid_amount` | FLOAT | DEFAULT 0 | Total payments received |
| `pending_amount` | FLOAT | DEFAULT 0 | total_amount âˆ’ paid_amount |
| `status` | TEXT | DEFAULT 'Sent' | Current status (see below) |
| `created_at` | TEXT | NOT NULL | Creation timestamp |

#### Invoice Statuses

| Status | Meaning |
|--------|---------|
| `Draft` | Not yet sent |
| `Sent` | Delivered to customer |
| `Due Soon` | Due within 3 days |
| `Due Today` | Due date is today |
| `Overdue` | Past due date |
| `Partially Paid` | Some amount received |
| `Paid` | Fully paid |
| `Disputed` | Customer has raised a dispute |
| `Escalated` | Escalated for collection |

### `payments`

Payment records against invoices.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | Payment identifier |
| `invoice_id` | UUID | FK â†’ invoices | Associated invoice |

| `amount` | FLOAT | NOT NULL | Payment amount |

| `payment_mode` | TEXT | DEFAULT 'UPI' | Payment method (UPI, Bank, Cash, etc.) |
| `reference_number` | TEXT | | Transaction reference ID |
| `created_at` | TEXT | NOT NULL | Record creation timestamp |

### `followups`

| `tone` | TEXT | DEFAULT 'Professional' | Message tone |
| `status` | TEXT | DEFAULT 'Drafted' | Status (Drafted, Sent, etc.) |
| `promised_payment_date` | TEXT | | Date customer promised to pay |
| `created_at` | TEXT | NOT NULL | Record creation timestamp |
