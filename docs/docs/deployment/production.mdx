# Production Deployment

Deploy PayGuard AI to production with a three-tier architecture: **Vercel** (frontend), **Railway** (backend), and **Neon** (database).

---

## Architecture Overview

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Vercel     │────▶│   Railway   │────▶│    Neon     │
│  (Frontend)  │     │  (Backend)  │     │ (Database)  │
│  React SPA   │     │  FastAPI    │     │ PostgreSQL  │
└─────────────┘     └─────────────┘     └─────────────┘
```

| Component | Platform | Why |
|-----------|----------|-----|
| Frontend | Vercel | Edge CDN, automatic deploys, free tier |
| Backend | Railway | Docker support, env vars, always-on |
| Database | Neon | Serverless PostgreSQL, branching, free tier |

---

## Step 1: Neon PostgreSQL Setup

1. Go to [neon.tech](https://neon.tech) and create an account
2. Click **Create Project**
   - **Project name:** `payguard-ai`
   - **Region:** Choose closest to your users (e.g., `aws-ap-south-1` for India)
3. Copy the connection string from the dashboard:

```
postgresql://neondb_owner:xxxx@ep-xxxxx.ap-south-1.aws.neon.tech/payguard_ai?sslmode=require
```

:::tip
Enable **SSL mode** (`sslmode=require`) for all production connections.
:::

### Create Tables

Run the schema migration against your Neon database:

```bash
cd backend
alembic upgrade head
```

Or use the seed script to populate demo data:

```bash
python seed.py
```

## Step 2: Railway Backend Deployment

1. Go to [railway.app](https://railway.app) and sign in with GitHub
2. Click **New Project → Deploy from GitHub Repo**
3. Select your `payguard-ai` repository
4. Railway will detect the `backend/` directory — set the **root directory** to `backend/`

### Set Environment Variables

Go to the **Variables** tab and add:

| Variable | Value |
|----------|-------|
| `DATABASE_URL` | `postgresql+asyncpg://neondb_owner:xxxx@ep-xxxxx.ap-south-1.aws.neon.tech/payguard_ai?sslmode=require` |
| `DB_NAME` | `payguard_ai` |
| `JWT_SECRET` | *(generate a strong random string)* |
| `JWT_ALGORITHM` | `HS256` |
| `JWT_EXPIRE_MINUTES` | `10080` |
| `CORS_ORIGINS` | `https://your-domain.vercel.app` |
| `EMERGENT_LLM_KEY` | `sk-emergent-xxxx` |
| `DEMO_EMAIL` | `demo@payguard.ai` |
| `DEMO_PASSWORD` | `demo123` |
| `LOG_LEVEL` | `INFO` |
| `LOG_DIR` | `./logs` |
| `SMTP_HOST` | `smtp.gmail.com` |
| `SMTP_PORT` | `587` |
| `SMTP_USER` | `your-email@gmail.com` |
| `SMTP_PASS` | `your-app-password` |
| `FRONTEND_URL` | `https://your-domain.vercel.app` |

### Deploy

Railway auto-deploys on push. To deploy manually, click **Deploy**.

### Verify

```bash
curl https://your-app.up.railway.app/health
```

Expected response:

```json
{
  "status": "healthy",
  "version": "1.0.0",
  "database": "connected"
}
```

## Step 3: Vercel Frontend Deployment

1. Go to [vercel.com](https://vercel.com) and sign in with GitHub
2. Click **Import Project** and select your repository
3. Configure:
   - **Framework Preset:** Create React App
   - **Root Directory:** `frontend/`
   - **Build Command:** `npm run build`
   - **Output Directory:** `build`

### Set Environment Variables

| Variable | Value |
|----------|-------|
| `REACT_APP_BACKEND_URL` | `https://your-app.up.railway.app` |

### Deploy

Click **Deploy**. Vercel will build and deploy automatically.

### Verify

Visit `https://your-project.vercel.app` and confirm the app loads.

## Custom Domain Setup

### Vercel (Frontend)

1. Go to **Project Settings → Domains**
2. Add your custom domain (e.g., `app.payguard.ai`)
3. Configure DNS:
   - **Type:** CNAME
   - **Name:** `app`
   - **Value:** `cname.vercel-dns.com`
4. SSL is auto-provisioned

### Railway (Backend)

1. Go to **Project Settings → Networking**
2. Click **Generate Domain** for a free `.railway.app` domain
3. For custom domains:
   - Add your domain (e.g., `api.payguard.ai`)
   - Configure DNS:
     - **Type:** CNAME
     - **Name:** `api`
     - **Value:** `your-app.up.railway.app`
4. Railway provides auto-SSL

---

## SSL/HTTPS

Both Vercel and Railway provide **automatic SSL certificates** via Let's Encrypt. No manual configuration needed.

For production, ensure:

- All API calls use `https://`
- The `FRONTEND_URL` in backend env vars uses `https://`
- The `REACT_APP_BACKEND_URL` in frontend uses `https://`

## Post-Deployment Checklist

- [ ] Database tables created (`alembic upgrade head`)
- [ ] Demo data seeded (optional: `python seed.py`)
- [ ] Health check returns `200 OK`
- [ ] Frontend loads at custom domain
- [ ] Login with demo credentials works
- [ ] API endpoints return data
- [ ] CORS configured for production frontend URL
- [ ] JWT_SECRET is a strong random string (not the dev default)
- [ ] SMTP configured for password reset emails
- [ ] Logs are accessible via Railway dashboard
- [ ] SSL certificate active on both domains
- [ ] Rate limiting configured (if needed)

:::caution
Never use the default `JWT_SECRET` from development in production. Generate a new one:

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```
:::
